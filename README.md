## Objetivo del repositorio:

Su **objetivo principal** es sentar una base de conocimiento inicial sobre el funcionamiento de Github a través de un enfoque eminentemente práctico. De la siguiente forma: 
- Creación de un archivo README para aplicar el formato markdown y entenderlo mejor
- Testeo de funciones de GIT ( como: commit, pull request, push, clone....

*Enlace a mi GitHub (https://github.com/Ex-1215/practicas-include13/edit/main/README.md)

---

### Conceptos clave:

- **Commit**: es cada vez que actualizas un fichero dentro del repositorio. Es una actualización de versión a nivel local.
- *Push*: sirve para descargar todos los commits de GitHub y enviarlos a Git. Se hace al final del dia.

---

## Convención de mensajes de commit

Un  buen mensaje de commit sigue el formato: tipo: descripcion breve

- feat: añado nueva funcionalidad
- fix: corrijo un error
- docs: actualizo documentacion
- chore: tareas de mantenimiento
